# 初赛高保真原型说明

对应 issue：`SHIKE-030`

## 原型目标

原型服务比赛演示，不做静态展示页。评委需要在 3 到 5 分钟内看到：

1. 手机原生入口：截图分享、相册导入或拍照触发。
2. AI 可解释理解：结构化字段、置信度、缺失字段。
3. 行动编排：日历、提醒、地图组合。
4. 执行回写：动作成功或降级提示。
5. 持续追踪：今日行动台和收件箱状态。

## 页面总览

| 页面 | 主要组件 | 用户动作 | AI/系统反馈 |
|---|---|---|---|
| Demo 控制台 | 手机首页预览、指标、演示路线、验收命令 | 打开 `prototype/demo.html` | 一页完成讲解、验收和交付包入口 |
| 今日行动台 | 今日三件事、即将截止、来源标签、隐私状态 | 点击课程或活动卡片 | 展示处理进度、截止风险和建议动作 |
| 截图导入卡 | 截图缩略图、意图标签、快捷动作 | 点“交给拾刻” | “识别到课程通知，建议安排日历+提醒” |
| 相机导办页 | 取景框、识别边框、场景提示、拍照按钮 | 拍海报并识别 | “识别到活动海报，置信度 0.91” |
| AI 解析确认页 | 字段表单、置信度、缺失字段、来源预览 | 修正时间或地点，点确认 | 生成可执行动作组合 |
| 行动编排页 | 日历、提醒、地图三块动作卡 | 勾选动作并一键安排 | 显示写入结果、权限降级提示 |
| 收件箱页 | 待确认、已安排、即将截止、已完成、已忽略 | 过滤、归档、执行 | 状态流转和来源归类 |
| 隐私与端云设置页 | 端侧优先、云侧增强、数据删除、权限管理 | 切换开关 | 展示当前处理策略和权限影响 |
| 活动详情页 | 活动卡、倒计时、地图入口、报名方式 | 打开报名方式或导航 | deeplink 或复制报名文本 |

## 核心链路一：课程通知

```text
截图分享/相册导入 -> 截图导入卡 -> AI 解析确认 -> 行动编排 -> 今日行动台
```

关键演示点：

- 截图导入卡必须清楚表达“用户把截图交给拾刻”，最近截图助手只作为通知式导入增强。
- 解析页必须展示“课程名、时间、地点、任务、置信度”。
- 行动编排页默认勾选日历和提醒，地图作为可选动作。
- 今日行动台必须出现“今晚 18:30 前查看新教室路线”。

## 核心链路二：活动海报

```text
相机导办 -> 海报结构化结果 -> 行动编排 -> 收件箱即将截止
```

关键演示点：

- 相机页要有海报识别框和底部动作建议，不做普通拍照页。
- 解析页必须有报名截止、活动时间、地点、主办方。
- 如果报名方式缺失，页面显示“缺失字段：报名链接”，但仍允许先创建提醒。
- 收件箱状态标记为“即将截止”，体现持续追踪。

## 页面文案规范

| 场景 | 推荐文案 | 禁用文案 |
|---|---|---|
| 入口 | 交给拾刻 | 开始 AI 聊天 |
| 解析 | 我识别到这是一条课程变更 | 我想我可能看懂了 |
| 确认 | 确认后安排 | 自动替你安排 |
| 动作 | 加入日历、提醒我、打开地图 | 授权全部自动执行 |
| 隐私 | 端侧优先，云侧增强，用户可控 | 数据全量上传 |

## 设计检查

- 每个页面都有明确用户动作，不出现纯展示页。
- 每个 AI 判断都配置信度或缺失字段，不做黑盒结论。
- 涉及日历/提醒/地图前必须经过用户确认。
- 所有页面文案围绕“行动闭环”，不写成收藏管理工具。
- 原型不依赖真实 vivo 系统权限，但清楚表达复赛实现边界。
