# MVP 范围收敛表

对应 issue：`SHIKE-010`

## 一句话边界

拾刻 MVP 只验证“青年场景碎片信息能否转成可确认、可执行、可追踪的行动卡”，不扩展成全能 AI 助手。

## 范围表

| 维度 | MVP 保留 | 验收标准 | 排除项 |
|---|---|---|---|
| 入口 | 截图导入、相机拍照 | 用户从截图或拍照进入同一条解析确认链路 | 分享面板、通知监听、语音导入、文档导入、桌面组件 |
| 场景 | 课程通知、活动海报 | 两类样例都有结构化字段、动作建议和收件箱状态 | 票据行程、兼职面试、电话摘要、会议纪要、复杂表单 |
| 动作 | 日历、提醒、地图 | 至少两个动作在复赛 MVP 中真实可用，第三个可通过 deeplink 或本地 mock 验证 | 联系人保存、回复草稿、表单预填、跨应用自动提交 |
| 持续层 | 收件箱、今日行动台 | 每条碎片信息具备待确认、已安排、即将截止、已完成、已忽略状态之一 | 主题图谱、长期知识库、校园模板市场 |
| AI 能力 | 场景判断、字段抽取、置信度、缺失字段、建议动作 | 输出符合 JSON Schema，用户可编辑字段后再执行 | 无边界自主 Agent、自动执行敏感动作 |

## 团队需求进入规则

任一新需求进入当前迭代前必须回答：

1. 是否属于截图导入或相机拍照入口。
2. 是否服务课程通知或活动海报。
3. 是否落到日历、提醒、地图三类动作之一。
4. 是否能在收件箱/今日行动台形成后续追踪。
5. 是否不依赖尚未确认的 vivo 系统重权限。

只要任一问题为否，该需求默认标为 P1/P2 或决赛增强，不进入 P0。

## 排除项清单

| 排除项 | 不进入首期原因 | 后续处理 |
|---|---|---|
| 全自动通知监听 | Android 权限、隐私审查和设备差异风险高 | 决赛增强，先以手动导入替代 |
| 语音/通话导入 | 与小V电话助手认知重合，且链路不如截图/拍照直观 | P2，作为会话闭环扩展 |
| 票据行程 | 场景有价值但会拉宽样例和动作复杂度 | P1 后半段或决赛模板 |
| 桌面组件/锁屏卡 | 展示强但系统权限和实现细节不确定 | 初赛画原型，复赛视权限补 |
| 跨应用自动代操作 | 高风险且容易显得不可控 | 只做可解释 deeplink 和用户确认 |
| 大型个人记忆库 | 容易偏离执行闭环，变成收藏工具 | 决赛做主题回看，不进入 MVP |

## 核心演示故事覆盖

| 故事 | 输入 | AI 解析 | 动作 | 追踪 |
|---|---|---|---|---|
| 课程通知 | 群截图：高数课临时改到 18:30，地点 B203 | 课程名、时间、地点、事项、置信度 | 加日历、提前提醒、地图导航 | 今日行动台显示“18:30 前查看新教室路线” |
| 活动海报 | 校园海报：AI 分享会，今晚报名截止，地点报告厅 | 活动名、时间、地点、报名方式、截止时间 | 加日历、截止提醒、地图导航 | 收件箱标为“即将截止” |
