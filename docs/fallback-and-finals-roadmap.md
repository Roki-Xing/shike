# 降级方案与决赛增强路线

对应 issue：`SHIKE-090`

## 降级矩阵

| 风险 | 触发信号 | 降级动作 | 用户可见结果 | 验证方式 |
|---|---|---|---|---|
| 模型失败 | API 超时、返回非 JSON、置信度低于 0.65 | 使用模板化抽取或进入待确认 | 显示“需要你确认字段”，不执行敏感动作 | 注入 unknown 样例，检查状态为待确认 |
| 权限拒绝 | 日历、通知权限未授权 | 写入本地行动卡和应用内倒计时 | 卡片留在收件箱，提示可稍后授权 | 禁用 calendar 权限跑 Spike |
| 网络不可用 | 云侧模型不可达 | 端侧 OCR + 本地规则初筛 | 只显示基础字段和手动提醒 | 断网或 mock API 失败 |
| 字段缺失 | 缺时间、地点、报名方式 | 标记 `missing_fields`，只允许安全动作 | 用户看到缺失项，仍可保存卡片 | 低质量样例回归 |
| 官方 API 变化 | BlueLM 或云测接口不可用 | 保持 `ModelAdapter`，切回 mock/OpenAI compatible | 上层流程不变，替换适配器即可 | 用三类适配方式返回同一 schema |
| 地图不可用 | 未安装地图或 deeplink 失败 | 复制地点、展示文本路线 | 不丢失地点信息 | 模拟 map 权限 false |

## 决赛增强路线

| 增强项 | 价值 | 依赖 | 不进入 MVP 的原因 |
|---|---|---|---|
| 端侧轻分类 | 更快识别截图是否值得处理，增强隐私 | 端侧模型或本地规则 | 初赛/复赛先用 mock 与规则证明链路 |
| 隐私脱敏 | 上传前移除手机号、姓名、地址等敏感字段 | OCR、NER、本地策略 | 需要更多样例和安全评估 |
| 桌面组件/锁屏卡 | 更像系统级体验，提升出圈效果 | Android Widget 或 vivo 系统能力 | 权限和适配成本高 |
| 智能体兼容层 | 对接蓝心智能体生态，扩展工作流 | 官方平台能力 | 不适合承担首期核心闭环 |
| 校园模板市场 | 支持课程、社团、竞赛、求职、旅行模板 | 模板治理和社区内容 | 需要运营和更多场景验证 |
| 主题记忆图谱 | 形成长期留存和主题回看 | 本地索引、聚类、搜索 | 容易偏向收藏器，需在闭环稳定后再做 |

## 决赛叙事升级

初赛和复赛讲“从截图到行动闭环”；决赛升级为：

**拾刻 OS 插件版：青年生活行动代理。**

增强演示应保持三条边界：

- 端侧优先、云侧增强、用户可控。
- 所有敏感动作前保留确认。
- 增强项服务执行闭环，不扩展成泛聊天助手。

## 故障注入清单

| 场景 | 注入方式 | 预期 |
|---|---|---|
| 模型失败 | 返回空对象或非 JSON | 进入待确认，记录模型失败 |
| 权限拒绝 | `calendar=false`, `map=false` | 降级为本地卡片和复制地点 |
| 网络不可用 | mock FastAPI 500 | 本地规则保底 |
| 字段缺失 | 输入低质量 OCR 文本 | `missing_fields` 非空 |
| 官方 API 变化 | 切换 Adapter 输出同一 schema | 上层测试不改 |
